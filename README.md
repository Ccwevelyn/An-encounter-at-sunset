# 日落相遇（An encounter at sunset）

校内恋爱匹配小站：@mpu.edu.mo 邮箱注册，填写档案与灵魂共鸣主观题，可进行随机匹配、缘分匹配、灵魂共鸣匹配，与匹配对象聊天。含三个聊天角色（最伟大最尊敬的导师 / 看不上你对象的朋友 / 知心姐姐）可闲聊。

---

## 功能概览

- **登录 / 注册**：邮箱须为 `@mpu.edu.mo`。注册需先获取邮箱验证码再填写昵称与密码；登录支持密码与验证码（4 位、5 分钟有效）。
- **档案**：性别、希望匹配的性别、学院、专业、生日、MBTI、恋爱经历、恋爱目的、城市、月花销、想恋爱指数等。
- **匹配**  
  - **随机匹配**：开启随机匹配状态后，在符合双向性别偏好的用户中随机一人。  
  - **缘分匹配**：按恋爱目的、城市、花销档位、想恋爱指数、经历、学位、MBTI 等规则筛选后择优随机。  
  - **灵魂共鸣**：5 道主观题一道一道作答后保存，在已答题且符合性别偏好的用户中，由 AI（DeepSeek）按契合度排序后匹配；未填主观题时点击会提示先填写。
- **聊天**：匹配列表、与对方聊天；列表前三位为三个固定角色（可与 TA 闲聊，支持 AI 回复）：**最伟大最尊敬的导师**（点拨引领、拓展思考）、**看不上你对象的朋友**（总嫌你对象配不上你、爱吐槽）、**知心姐姐**（温柔和蔼），需配置 DeepSeek 时由 AI 生成回复。
- **数据库**：支持 SQLite（默认）与 PostgreSQL（设置 `DATABASE_URL` 时自动切换）。

## 关于三个聊天角色（导师 / 看不上你对象的朋友 / 知心姐姐）

三个角色均为**固定系统提示（system prompt）**驱动，没有用你的聊天记录训练模型。配置了 `DEEPSEEK_API_KEY` 后，每次发消息会把对应角色的提示 + 你的话发给 DeepSeek 生成回复；未配置时使用内置简短规则回复。角色设定：**最伟大最尊敬的导师**庄严智慧、善于点拨与拓展思考；**看不上你对象的朋友**总嫌你对象配不上你、爱吐槽泼冷水但本质关心你；**知心姐姐**温柔、包容、善解人意。

## 查看数据库

- **本地用 SQLite 时**：在浏览器打开 `/admin?secret=你的ADMIN_SECRET`（环境变量 `ADMIN_SECRET` 需与 URL 中 `secret` 一致），可查看/编辑 users、profiles、matches、messages 等表。
- **部署用 PostgreSQL 时**：不提供 `/admin`，在 Render（或所用平台）控制台里打开该 PostgreSQL 实例，用其自带的数据库管理或外部客户端连接 `DATABASE_URL` 查看。

## 邮件与费用（Resend）

发登录/注册验证码使用 Resend。Resend **免费额度**：每月 3000 封、每日约 100 封，无需绑卡；超出后才需付费。本项目的验证码用量一般都在免费额度内。

## 技术栈

- 前端：React 18、React Router、Vite  
- 后端：Node.js、Express、SQLite / PostgreSQL、Resend（验证码邮件）、DeepSeek API（可选）
